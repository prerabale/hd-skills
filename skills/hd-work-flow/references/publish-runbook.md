# Picasso 发布 Runbook（QA）

## 目标
将指定迭代发布到 QA，并确认流水线已真正触发。

## 操作步骤
**必须通过 Picasso 迭代详情页的发布功能进行发布，不能通过其他途径（如手动 git 操作或流水线页面直接触发）。**

1. 在迭代详情页点击 **发布**。
2. 确认弹窗标题为 **选择发布模版**。
3. 读取可选模板（radio）。
4. 选择模板：
   - 默认 QA：`disconf发布qa`
   - 非多域名 QA：`disconf发布qa 非多域名`
5. 点击弹窗 **确定**（关键步骤）。
6. 返回详情页，检查“正在执行流水线”是否新增本次记录。

## 成功判据
满足任一：
- 新流水线卡片包含 `qa` 与所选模板名；
- 出现本次时间戳的新执行记录。

## 常见误区
- 只点了“发布”，没点弹窗“确定”；
- 选错模板（dev/prod）；
- 弹窗消失但没有新增流水线记录却误报成功。

## 失败重试
- 重新打开发布弹窗并再次确认模板；
- 刷新后再尝试 1 次；
- 仍失败：回报当前状态（模板、时间、错误提示）。
