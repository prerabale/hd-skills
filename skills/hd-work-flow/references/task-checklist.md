# HD Work Flow 执行清单

## 启动前
- [ ] 已确认模式：迭代 / 修改
- [ ] 迭代模式下已拿到文档链接或摘要
- [ ] 已推测并确认项目（1个或多个）

## Picasso 侧
- [ ] 需求存在且字段完整（名称/策划人/文档地址）
- [ ] 迭代存在且应用匹配项目
- [ ] 已记录 target branch（系统分配分支）

## 开发侧
- [ ] **严禁在迭代分配的 target 分支直接修改提交**
- [ ] 在 worktree 开发分支中进行开发（不是 target 分支）
- [ ] 代码已提交并推送到 worktree 分支（source branch）
- [ ] 使用 git push push options 创建 MR，提交用户审批

## 发布侧（如需要）
- [ ] 已点击“发布”
- [ ] 已在弹窗选择正确模板
- [ ] 已点击弹窗“确定”提交
- [ ] 已看到“正在执行流水线”的新增记录

## 收尾
- [ ] **必须：发送任务完成通知**（包含需求/迭代信息、分支、MR状态、发布状态、后续建议）
- [ ] 已把关键结果回报用户（分支、MR、发布状态）
- [ ] 用户确认后再清理 worktree
- [ ] 未删除远端分支（除非用户明确要求）
